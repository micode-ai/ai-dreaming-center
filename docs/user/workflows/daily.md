# Один день жизни инструмента

Типичный workflow по dependancу DC, утром-в-обед-вечером. Если ты используешь DC активно — ты будешь делать что-то такое каждый день.

## Утро: проверка после ночи

Открываешь DC по адресу http://localhost:8086 (или где у тебя крутится). Тебя встречает agg dashboard на `/`.

1. **Top-line метрики**: посмотри на 6 цифр сверху:
   - Успешные за неделю: должно расти каждый день.
   - Сбой / Таймаут: должно быть мало, или хотя бы стабильно.
   - running now: обычно 0 утром (ночные cron'ы должны были закончиться к утру). Если что-то висит — что-то залипло.
   - Тех-долг / Идеи: общая сумма по проектам.

2. **Карточки проектов**: пробегись по карточкам — у каждого свои `ok / fail / timeout / running`. Если у одного проекта вдруг все timeouts — это red flag.

3. **Активные сессии (правый сайдбар)**: если что-то ещё running — кликни, посмотри, что висит. Долгая сессия может быть нормой (cascade run), а может — залипший процесс.

5–10 минут на morning sweep.

## До обеда: разбор результатов

Заходишь в активный проект (тот, в котором сейчас работаешь). Например `/p/my-app/`.

1. **Recent sessions** на dashboard: 10–20 последних сессий. Смотри:
   - Какие агенты учились ночью.
   - У всех ли `success`, или есть `failed`/`timeout`.
   - Когда последний раз был запущен главный для тебя агент (например `vera`)? Если давно — повысь tier до P1 на rotation, чтобы cron взял быстрее.

2. **Если есть failed** — кликни на запись или иди на `/p/{slug}/live` (даже если уже не running, JSONL остался в `~/.claude/projects/`). Понять причину:
   - Не нашёл агент-файл.
   - Rate-limit от API.
   - Bad prompt в slash-command.
   - И т.п.

3. **Конспекты** на `/p/{slug}/notes` — пройди по новым md-файлам. Хорошие конспекты — добывай в Obsidian, мусор — игнорь.

15–20 минут на review.

## После обеда: подкрутить custom topics

Полезный момент: если за утро у тебя сложились задачи на сегодняшнюю-завтрашнюю работу, можно «попросить» агентов изучить конкретные темы за следующую ночь.

1. Открой `/p/{slug}/kanban`.
2. Добавь 1–2 новых topic'а:
   - Title — короткое описание.
   - target_agents — конкретно тот, кому полезно. Или `*` если всем.
   - question — what specifically.
   - why_important — почему сейчас.
3. Save.

Это подмешается в prompt nightly self-study этой ночью.

Опционально пройди по существующим custom topics: те, что уже неактуальны — `delete`.

5 минут.

## Под вечер: tech-debt и ideas

Часов в 17–18, перед уходом, обычно полезно:

1. **Findings** на `/p/{slug}/findings` — посмотри список tech-debt items. Закрой (close) те, которые ты сегодня сделал. Удали (delete) дубликаты или ошибочные.

2. **Tech-Debt aggregate** на `/p/{slug}/tech-debt` — посмотри карточку «По статусу». Сколько open vs closed? Если open > 50 — пора триаж.

3. **Ideas** на `/p/{slug}/ideas` — посмотри backlog. Если есть очевидно полезная идея — нажми `→ Jira` чтобы создать тикет (см. [`jira-integration.md`](jira-integration.md)).

4. **Sidecar findings** на `/p/{slug}/sidecar-findings` — если у тебя крутятся reviewer-агенты (vera/svetlana), посмотри, что они нашли сегодня. Critical / high — приоритет.

15 минут.

## Перед уходом: проверить cron

1. **Глобально**: на `/settings` посмотри:
   - `cron_enabled = true` (default).
   - `cron_expression = 0 3 * * *` (default 3 утра) или то время, которое тебе подходит.
   - `agents_per_night = 3` (или сколько ты хочешь).

2. **Per-project** (если есть особенности): на `/p/{slug}/settings` проверь, что override'ы для cron_enabled / cron_expression — те, что ты ожидал.

3. **Default project**: на `/projects` убедись, что хотя бы один проект помечен `★`. Это не обязательно, но удобно.

4. **Активные сессии**: убедись, что нет «забытых» running-сессий. Если есть — Kill через `/p/{slug}/live`.

5 минут.

Если всё в порядке — закрывай браузер, ноутбук может уходить в sleep (KeepAwake уже разрешает, нет running-сессий). Cron сам стартанёт ночью, к утру у тебя будет свежая порция данных.
