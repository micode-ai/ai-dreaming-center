# Добавить новый проект в работающий instance

Допустим, DC уже установлен и работает. У тебя уже есть N проектов в registry. И вот ты создал новый репо, и хочешь его подключить.

## Шаг 1. Подготовь проект на диске

DC ожидает, что новый проект — это папка под `projects_root`. Пример: `projects_root = D:\Work\micode\`, новый проект — `D:\Work\micode\my-new-project\`.

Минимум для распознавания:
1. Папка существует.
2. (Желательно) `.git/` внутри — DC использует это как маркер проекта.
3. (Желательно для self-study) `.claude/agents/` с хотя бы одним md-файлом.

Если у тебя только git-init'нутый репо без агентов — DC его всё равно распознаёт, но Rotation будет пуст пока ты не положишь md-файлов.

Можешь скопировать starter-kit:
```
git clone https://github.com/RsCloud2022/agent-team-starter-kit.git temp-kit
xcopy /E /Y temp-kit\.claude D:\Work\micode\my-new-project\.claude
rmdir /S /Q temp-kit
```

## Шаг 2. Импорт через `/projects`

DC сервер уже работает (uvicorn крутится).

1. Открой http://localhost:8086/projects (или прицепляйся к работающему).
2. Пролистай вниз до блока «Импорт из projects_root».
3. В input'е увидишь текущий `projects_root` (например `D:\Work\micode`). Можешь оставить — или переопределить на одноразовое сканирование (если новый проект где-то ещё).
4. Нажми синюю `Просканировать и импортировать новые`.

DC просканит указанный путь. Уже зарегистрированные проекты (по `working_dir` matching) пропустит. Новые — добавит:
- slug = lowercase(имя папки), `_` заменено на `-`.
- label = имя папки as-is (с capitalisation первой буквы).
- working_dir = абсолютный путь.
- enabled = true.
- is_default = false.

После redirect'а на `/projects` ты увидишь новую строку в таблице.

Готово — проект импортирован. Cron-jobs автоматически зарегистрировались (`nightly_learning_{slug}` запустится по cron_expression сегодня ночью).

## Шаг 3. Настройки per-project (опционально)

Если для нового проекта нужны settings отличные от global:

1. Открой `/p/{slug}/settings`.
2. Найди ключ, который хочешь override'нуть. Часто:
   - `tech_debt_dir`, `product_ideas_dir`, `wiki_dir` — пути специфичны для этого проекта.
   - `cron_expression` — если этот проект надо учить в другое время.
   - `agents_per_night` — если у этого проекта много агентов и хочешь больше за ночь.
   - `jira_project_key` — если другой Jira-проект.
3. Кликни radio `Override`, впиши значение в поле, кликни Save.

Изменения сохранятся в `project_settings` таблице, применятся сразу. Рестарт не нужен.

## Шаг 4. Проверка cron-jobs

Хочется убедиться, что nightly cron реально зарегистрирован.

В UI пока нет страницы «список cron-jobs» (TODO для будущих волн). Косвенные способы:
- Перезапусти uvicorn в шумном режиме (`--log-level debug`) — APScheduler в логах напишет: «Adding job nightly_learning_my-new-project trigger=cron(0 3 * * *) ...».
- В `data/dreaming.db-jobs` (или встроенная APScheduler-jobstore — путь зависит от конфигурации) можно посмотреть jobs через SQL.
- Проще: дождись cron-expression. Если в 3:00 ночью запустилась сессия для нового проекта — значит, всё работает.

Если cron не зарегистрировался автоматически — рестарт DC обычно решает: lifespan заново читает projects table и регистрирует jobs.

## Шаг 5. Первая сессия

Чтобы убедиться что всё работает end-to-end, не дожидаясь ночи:

1. Открой `/p/{slug}/rotation`.
2. Если есть md-файлы в `.claude/agents/` — увидишь таблицу. Если нет — создай хотя бы один и обнови страницу.
3. Нажми `Start session` рядом с любым агентом.
4. Тебя редиректит на `/p/{slug}/live`. Смотри, как идёт сессия.
5. После завершения — на dashboard `/p/{slug}/` будет одна запись.

Готово. Проект подключён, cron работает, ручной запуск работает.

## Удаление, если что-то пошло не так

Если ты импортировал по ошибке (не та папка, неправильный slug):

1. На `/projects` найди строку.
2. Нажми красный `Delete`.
3. JS-prompt спросит slug — введи точно как в колонке.
4. Запись удаляется, cron-jobs unregister'ятся.

Если хочешь только переименовать slug — удали и заимпортируй заново. UI редактирование slug не поддерживает.

Если хочешь временно отключить (без потери истории):
- Кнопка `Disable` рядом с `Delete`. Cron-jobs unregister'ятся, но данные сохраняются. `Enable` обратно — всё восстановится.
