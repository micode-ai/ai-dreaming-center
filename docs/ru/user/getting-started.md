## Предусловия

- Python 3.10+ установлен (проверь: `python --version`).
- `claude` CLI установлен и доступен из `PATH` (проверь: `claude --version`). На Windows должен быть `claude.cmd`, на macOS/Linux — обычный `claude`.
- Хотя бы одна папка с проектами под `projects_root`. У большинства это `D:\Work\micode\` или `~/Projects/` — внутри лежат отдельные репозитории, желательно с `.claude/agents/` для self-study.

## Шаг 1. Установка (3–5 минут)

Открой терминал и выполни (PowerShell/bash, неважно):

1. Склонируй репозиторий:
   ```
   git clone <repo-url>
   cd ai-dreaming-center
   ```
2. Создай и активируй venv:
   ```
   python -m venv .venv
   .\.venv\Scripts\Activate.ps1     # PowerShell
   source .venv/Scripts/activate    # bash
   ```
3. Установи пакет в editable-режиме:
   ```
   pip install -e .
   ```
   На этом шаге pip потянет FastAPI, Uvicorn, APScheduler, aiosqlite, sse-starlette, jinja2, pydantic-settings, httpx и pyyaml. Минут 1–2.

## Шаг 2. Запуск сервера (1 минута)

```
python -m uvicorn dreaming.main:app --port 8086
```

В консоли увидишь Uvicorn-логи: `INFO: Started server process`, `INFO: Application startup complete.` — сервер живой.

При первом старте `config.yaml` ещё не существует — middleware redirect'ит любой запрос на `/setup`.

Открой в браузере: `http://localhost:8086`. Тебя перекинет на `http://localhost:8086/setup`.

## Шаг 3. Setup wizard (3–5 минут)

На странице `/setup` ты увидишь заголовок «Первичная настройка» и форму из двух блоков:

**Блок 1 — «Глобальные настройки»** с тремя полями:
- `claude_path` — путь к Claude CLI. По умолчанию там стоит `"claude"`. Если у тебя нестандартное место — впиши абсолютный путь, например `C:\Users\me\AppData\Roaming\npm\claude.cmd`.
- `projects_root` — корневой каталог, где DC будет искать твои проекты. По умолчанию там пусто. Впиши абсолютный путь, например `D:\Work\micode`.
- `default_locale` — язык интерфейса. Выбери `Русский` или `English` из dropdown.

**Блок 2 появится только после сканирования.** Внизу формы есть синяя кнопка `Просканировать projects_root`. Нажми её.

Страница перезагрузится. Если в `projects_root` есть подпапки — увидишь таблицу с заголовком «Найденные проекты в {path}» и колонками:
- `включить` — checkbox (по дефолту отмечен только если в папке есть `.claude/`).
- `по умолч.` — radio-кнопка (выбери проект, который будет default'ом).
- `slug` — короткое имя (можно поправить).
- `label` — человекочитаемое имя.
- `путь` — абсолютный путь, readonly.
- `.claude` — галочка `✓` если папка `.claude/` есть в этом проекте.

Под таблицей: «X проектов найдено. Включённые — будут импортированы.»

Сделай:
1. Сними чекбоксы с тех проектов, которые тебе сейчас не нужны (они потом импортируются через `/projects`).
2. Выбери один default-радиобокс — это проект, который откроется по дефолту при заходе на `/p/`. (Можешь оставить и первый.)
3. Нажми синюю `Сохранить и импортировать`.

После клика страница пропадёт, тебя перекинет на `/`.

Если сканирование выдало красную плашку `Ошибка: ...` — значит, путь не существует или нет прав. Проверь, что `projects_root` указывает на существующую директорию.

## Шаг 4. Проверка (1 минута)

На `/` ты должен увидеть:
- Заголовок «AI Dreaming Center» + строку «N active projects».
- Шесть top-line метрик: Успешные / Сбой / Таймаут (за неделю), running now, Тех-долг, Идеи продукта. Все нули.
- Карточки проектов (по одной на каждый импортированный) с кнопками-ссылками на `/p/{slug}/`.
- Справа сайдбар «Активные сессии» — пустой.

Кликни по любой карточке → попадёшь на `/p/{slug}/`.

В шапке должен быть `<select>` со всеми проектами и кнопка `EN/RU` справа.

## Шаг 5. Первая сессия (3–5 минут)

1. На странице проекта переключись на вкладку `Ротация`.
2. Если в `.claude/agents/` есть md-файлы — увидишь таблицу: agent / tier / enabled / last_studied / кнопка `Start session`. Сверху строка вида «N agents in DB; M on disk in {working_dir}/.claude/agents/».
3. Если таблица пустая («0 agents in DB; 0 on disk») — у проекта пока нет агентов. Создай хотя бы один `.claude/agents/test.md` (можно просто `# Test agent\nЯ просто читаю файлы.`) и обнови страницу.
4. Нажми синюю кнопку `Start session` рядом с любым агентом.
5. Тебя редиректит на `/p/{slug}/live`. Там увидишь чёрный pre-блок, в который потоково льётся stdout claude.
6. Через несколько секунд–минут (зависит от модели и агента) поток закончится фразой `[stream ended]`. Это значит сессия завершилась.

Возвращайся на `/p/{slug}/` — должна появиться запись в «Последние сессии» со статусом `success` (или `failed` / `timeout` если что-то пошло не так).

Если статус `success` и в конспекте написан `note_path` — заходишь в `Конспекты`, кликаешь по файлу, читаешь.

## Troubleshooting

**`claude` CLI не найден** — uvicorn в логе пишет `FileNotFoundError`. Проверь `claude --version` в той же шелл-сессии. Если работает там — впиши абсолютный путь в `claude_path` через `/settings` или env var `DC_CLAUDE_PATH`.

**Порт занят** — uvicorn падает с `Address already in use`. Запусти на другом порту: `python -m uvicorn dreaming.main:app --port 8087` (не забудь поправить `port` в Settings, если шлёшь callback'и из claude обратно в DC).

**Setup wizard говорит «Ошибка: ... not found»** — `projects_root` указывает на несуществующую папку. Создай её (`mkdir D:\Work\micode`) или впиши другой путь.

**Сканер не нашёл ни одного проекта** — внутри `projects_root` нет подпапок. Создай хотя бы одну, в неё положи `.git/` или `.claude/agents/` (это маркеры для распознавания), пересканируй.

**Первая сессия `failed` с `error_message`** — открой `/p/{slug}/live`, прокрути вниз — увидишь stderr или последние строки stdout. Часто причина — Claude не понял prompt (агент-файл пустой или сломан) или нет API key.

**Сессия в статусе `running` но в `/live` ничего нет** — процесс Claude уже убит OS, а DB row застряла. Через 5 минут reconcile job её закроет. Можешь подождать или ручками `/projects/{id}/toggle` (disable+enable) чтобы сбросить.
