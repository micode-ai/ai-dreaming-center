# Ротация агентов

`/p/{slug}/rotation` — таблица со списком всех агентов проекта. Здесь ты управляешь тем, кого crontab берёт ночью, какой у каждого приоритет, и можешь запустить любого вручную.

## Открытие страницы

Из шапки проекта кликни вкладку `Ротация` (англ. `Rotation`). URL: `/p/{slug}/rotation`.

При первом открытии DC просканирует папку `{working_dir}/.claude/agents/`, найдёт все md-файлы, и автоматически создаст для каждого запись в `agent_learning_rotation` с `tier=2`, `enabled=true`. Это синхронизация: новые файлы попадают в БД при следующем visite.

## Что показывает таблица

Сверху строка вида: «N agents in DB; M on disk in {working_dir}/.claude/agents/».

Если N = M — синхронизировано. Если N < M — кто-то добавил md-файл вне DC и пока не открывал rotation. После открытия страницы — синхронизируется.

Колонки:
- **agent** — имя агента (без `.md`), моноширинный шрифт.
- **tier** — `<select>` с тремя опциями: `P1` / `P2` / `P3`. Текущий выбран.
- **enabled** — кнопка-toggle: `✓` если enabled, `—` если disabled.
- **last_studied** — timestamp последней успешной self-study сессии или `—` если никогда не было.
- (последняя колонка) — кнопка `Start session` (синяя) либо текст `running…` (амбер) если уже идёт.

## Откуда берутся агенты

Пути:
1. **Filesystem scan** — DC сканит `{working_dir}/.claude/agents/*.md` каждый раз при открытии страницы. Имя файла без расширения становится `agent_name`.
2. **Создание новой записи** — если такого `(project_id, agent_name)` нет в БД, добавляется с tier=2.
3. **Удаление** — если md-файл удалён с диска, запись в БД остаётся (но в таблице её не покажет, потому что filtered по filesystem). Чтобы удалить навсегда — DELETE напрямую в БД.

Если ты хочешь чтобы агент появился в DC:
- Создай файл `D:\Work\micode\my-app\.claude\agents\my-agent.md`.
- Положи в него хотя бы заголовок и описание.
- Открой `/p/my-app/rotation` — увидишь.

Имена в DC должны совпадать с именами файлов **точно** (case-sensitive на Linux/macOS).

## Изменение tier

Tier определяет приоритет агента в nightly cron:
- **P1 (high)** — высокий приоритет. Берётся первым при равном `last_studied_at`.
- **P2 (normal, default)** — обычный приоритет.
- **P3 (low)** — низкий. Только если нет P1/P2 кандидатов.

Алгоритм nightly cron: `ORDER BY last_studied_at ASC NULLS FIRST, tier ASC`. То есть:
1. Сначала те, кого вообще не учили (NULL last_studied_at).
2. Потом самые давно не учившиеся.
3. При равном времени — P1 раньше P2 раньше P3.

Как менять:
1. В колонке `tier` нажми на `<select>`.
2. Выбери `P1` / `P2` / `P3`.
3. Form auto-submit'нется (`onchange="this.form.submit()"`). Страница перезагрузится с новым значением.

Никакого Save-кнопки — каждый change мгновенно сохраняется.

## Toggle enabled

Если `enabled = ✓` — агента включает в свой выбор nightly cron.
Если `enabled = —` — cron его пропускает. Но ручной `Start session` работает в любом случае.

Чтобы переключить:
1. Кликни на кнопку с `✓` или `—`.
2. POST на `/p/{slug}/rotation/toggle` отправляется.
3. Страница перезагрузится с обновлённым значением.

Используй disable когда:
- Агент сломан и его конспекты — мусор.
- Тебе временно не нужно тратить токены на этого агента.
- Агент в рефакторинге, и self-study даст ложные insight'ы.

## Start session (кнопка)

Синяя кнопка `Start session` рядом с агентом — POST на `/p/{slug}/rotation/start/{agent}`.

Что происходит:
1. DC проверяет, не запущена ли уже сессия для этого агента в этом проекте. Если да — отдаёт 409.
2. Создаёт запись в `agent_learning_sessions` со статусом `running`.
3. Резолвит claude_path (на Windows ловит `claude.cmd`).
4. Спавнит claude с командой `{self_study_command} {agent}`.
5. Поднимает watchdog на `timeout_minutes`.
6. Подписывает live-log streamer на stdout.
7. Редиректит браузер на `/p/{slug}/live`.

Дальше — см. [`live-log.md`](live-log.md).

Если в момент клика глобальный лимит `max_concurrent` достигнут — DC отдаст ошибку (нужно подождать).

## Если агентов нет

Сценарий: ты только что создал проект, в нём нет `.claude/agents/`.

На странице rotation увидишь пустую таблицу и строку «0 agents in DB; 0 on disk in {working_dir}/.claude/agents/».

Что делать:
1. Создай папку: `mkdir D:\Work\micode\my-app\.claude\agents`
2. Положи туда хотя бы один md: `echo "# Test agent" > D:\Work\micode\my-app\.claude\agents\test.md`
3. Обнови страницу — агент появится.

Альтернатива: используй agent-team-starter-kit. Это отдельный репозиторий с готовым набором агентов (vera, svetlana, silent-failure-hunter, etc.) — клонируешь его поверх своего проекта, и сразу получаешь полный набор.
