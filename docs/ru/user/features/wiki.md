## Что такое wiki в DC

Wiki — это набор markdown-файлов в `wiki_dir` (`{working_dir}/wiki/` по дефолту), описывающих доменную структуру проекта. Один файл — один domain (auth, billing, ui-shell, etc.). Используется агентами в orchestration: Roman при декомпозиции цели читает domain-файлы чтобы понять, в какой части кодовой базы цель.

Конкретный формат wiki зависит от стартеркита/конвенций проекта. DC просто показывает количество domain-файлов и их имена — содержимое не парсит и не редактирует.

## Что показывает страница

Открой `/p/{slug}/wiki`. Логика:

**Сценарий 1: `wiki_dir` не настроен.**
- Амбер-плашка «Каталог wiki не настроен.»
- Ссылка на settings: «Пропиши `wiki_dir` в Settings или через CLI: `set_setting('wiki_dir', '...')`.»

**Сценарий 2: настроен, но не существует.**
- Серый текст: «Wiki не найдена. Каталог `{wiki_dir}` не существует.»
- Подсказка: «Можно запустить bootstrap прямо из UI — Claude CLI выполнит `/wiki-bootstrap` в рабочем каталоге проекта.»
- Синяя кнопка `Run /wiki-bootstrap (запустит claude)` (POST-форма).

**Сценарий 3: существует и непуст.**
- Белая карточка со статусом:
  - «Источник: `{wiki_dir}`».
  - Большим шрифтом: «N domains».
  - Если в wiki_dir есть md-файлы — `<details>` блок «показать первые N» с моноширинным списком имён файлов.

Имена показываются без расширения и без пути — только короткое имя domain'а.

## Run /wiki-bootstrap

Кнопка `Run /wiki-bootstrap` появляется только в сценарии 2 (настроен, но не существует).

Что происходит при клике:
1. POST на `/p/{slug}/wiki/bootstrap`.
2. DC резолвит claude_path и спавнит `claude /wiki-bootstrap` с `cwd={working_dir}`.
3. Создаёт row в `agent_learning_sessions` (с агент-name `_wiki-bootstrap` или подобным).
4. Подписывает live-log streamer.
5. Редиректит браузер на `/p/{slug}/live`.

Дальше — в live-логе видишь, как Claude:
1. Читает `.claude/agents/` чтобы понять команду агентов.
2. Сканит кодовую базу.
3. Определяет domains (по структуре каталогов / module-границам).
4. Создаёт `wiki_dir` если нет.
5. Пишет один md-файл на domain.

Это занимает обычно 10–20 минут. По окончании можешь обновить `/wiki` — увидишь сценарий 3 с domain-count > 0.

## После bootstrap'а

После успешного bootstrap'а:
- В `wiki_dir` лежат md-файлы по доменам.
- Roman при orchestration runs может их читать (если так написана его инструкция).
- `/p/{slug}/wiki` показывает count и список.

Доменной wiki не редактируется через UI. Если хочешь добавить/изменить domain:
- Открой md-файл вручную (Obsidian, VS Code, любой редактор).
- Или запусти агент `/wiki-update` (если у тебя в стартеркит'е такая команда есть).

## Если wiki уже есть

Если `wiki_dir` существует и в нём md-файлы — кнопки `Run /wiki-bootstrap` **нет**. Это сделано умышленно: bootstrap может перезатереть существующее.

Чтобы re-bootstrap:
1. Удали `wiki_dir` руками (или переименуй в `wiki.bak`).
2. Открой `/wiki` — теперь сценарий 2.
3. Жми кнопку.

## Если wiki_dir не настроен

По дефолту `wiki_dir` пустой. Чтобы:
1. `/p/{slug}/settings` → найди ключ `wiki_dir` (обычно в группе Paths).
2. Override → впиши, например, `D:\Work\micode\my-app\wiki\` (можно с trailing-slash, можно без).
3. Save.
4. Вернись на `/wiki` — сценарий поменяется.

Если хочешь чтобы wiki хранилась не в проекте, а в Obsidian-vault'е — впиши путь к vault'у в `wiki_dir`. Это работает: DC просто читает md-файлы из указанного каталога, не наследует от структуры проекта.
