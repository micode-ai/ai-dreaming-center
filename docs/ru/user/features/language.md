# Язык интерфейса

DC поддерживает русский (default) и английский. Переключается одной кнопкой в шапке.

## Где переключатель

В шапке справа — небольшая кнопка `EN` (если ты на русском) или `RU` (если на английском). Кликни — язык переключится.

Технически:
- POST на `/locale` с параметром `locale` (`ru` или `en`).
- DC ставит cookie `dc_locale` на 1 год.
- Редирект обратно на ту же страницу (через hidden `next` поле).

Кука читается на каждом request'е, и Jinja-фильтр `t()` подставляет правильный перевод. Без перезагрузки страницы изменение не происходит — нужен redirect.

## Что переводится

UI shell — заголовки, кнопки, метки, navigation:
- Заголовки страниц («Проекты» / «Projects»).
- Названия вкладок («Ротация» / «Rotation», «Конспекты» / «Notes», «Тех-долг» / «Tech debt»).
- Кнопки (`Disable` / `Enable`, `Просканировать` / «Scan», `Сохранить` / `Save`).
- Метки колонок (`agent`, `status`, etc. — но многие моноширинные английские оставлены as-is).
- Тексты-плейсхолдеры (например, на пустом state'е: «Сессий ещё нет» / «No sessions yet»).
- Сообщения об ошибках с UI-side.

Файлы переводов:
- `dreaming/i18n/messages_ru.json` — русский.
- `dreaming/i18n/messages_en.json` — английский.
- Структура: flat ключ → значение. Например `"p.dashboard": "Панель"` (ru) или `"Dashboard"` (en).

## Что не переводится

- **Markdown content** твоих агентов, конспектов, tech-debt, ideas. DC показывает их as-is. Если ты пишешь note по-английски — он останется английским независимо от UI-locale.
- **Имена агентов** (`vera`, `svetlana`, `roman`) — это идентификаторы, не переводятся.
- **Slug'и проектов** — то же.
- **Технические ключи настроек** (`claude_path`, `cron_expression`) — code identifiers, не должны меняться.
- **Stdout claude'а** в `/live` — что Claude вывел, то и видишь. На каком языке он генерирует — зависит от prompt'ов твоих агентов.
- **JSON-ответы API** — техника.

Идея простая: chrome переведён, content — не трогается.

## Default locale

При первом заходе (когда куки `dc_locale` ещё нет) DC берёт default из настроек:
- Глобально — `default_locale` ключ (default `ru`).
- В setup wizard'е есть dropdown «Русский / English» — твой выбор пишется в `config.yaml` как `default_locale`.

Чтобы поменять default позже — открой `/settings` → найди `default_locale` → впиши `ru` или `en` → Save.

## Добавить свой язык

Скажем, хочешь немецкий. Шаги:

1. Создай файл `dreaming/i18n/messages_de.json` с тем же набором ключей что в `messages_ru.json` / `messages_en.json`. Переведи значения.
2. Зарегистрируй язык в коде i18n (см. `dreaming/i18n/__init__.py` или подобный — там список supported locales).
3. Добавь option в setup wizard и в шапку (currently они hardcoded для ru/en).
4. Перезапусти uvicorn.

Это код-чейндж, не configuration. Если ты не разработчик — лучше fork репо, добавь язык, открой PR.

## Сообщить о неточности

Если в каком-то переводе криво — открой issue в репозитории с:
- Английским/русским исходником.
- Текущим переводом.
- Предложенным улучшением.

Часто переводы делались автоматически (LLM-translation) и могли пропустить нюансы. Любые правки welcome.

Если ты сам исправляешь — открой PR. Файлы малые (текущий `messages_ru.json` — ~40 строк), парсити просто, конфликтов не бывает.
