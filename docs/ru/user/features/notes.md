# Конспекты

`/p/{slug}/notes` — браузер по конспектам, которые агенты пишут после self-study сессий. Простая read-only страница: список файлов + просмотр raw-content.

## Что показывает страница

Открой `/p/{slug}/notes`. Сверху — строка вида: «Источник: `{learning_notes_dir}`». Если каталога не существует — рядом красным «(каталог не существует)».

Если файлы есть — таблица с двумя колонками:
- **путь** — относительный путь от `learning_notes_dir`. Кликабельная ссылка (синяя, моноширинная).
- **размер** — bytes (раздел справа).

Например:
```
2026-05-09-vera.md           4231
2026-05-08-svetlana.md       3892
old/2026-04-15-team-lead.md  5021
```

Если поддиректории внутри `learning_notes_dir` есть — DC обходит рекурсивно. Файлы фильтруются по расширению `.md` (только markdown).

## Откуда берутся файлы

Конспект — это markdown, который агент пишет в конце self-study сессии. По умолчанию `learning_notes_dir` = `{working_dir}/.claude/agents/learning-notes/`. Можно override'нуть в settings (`learning_notes_dir`).

Имя файла — на усмотрение агента (то, что прописано в `/self-study` slash-команде starter-kit'а). Обычно это `{date}-{agent}.md` или `{date}/{agent}.md`.

После завершения сессии:
1. Claude пишет файл (через свой Write tool).
2. Опционально через callback `POST /api/session/finish` отдаёт DC путь — DC сохраняет в `agent_learning_sessions.note_path`.
3. На `/p/{slug}/dashboard` в recent sessions ты видишь имя агента и (если note_path есть) — его как кликабельный линк.

В таблице `/notes` видны **все** md-файлы каталога, не только новейшие. Это удобно: можешь вернуться к старому конспекту, если что-то нужно вспомнить.

## Просмотр raw-content

Клик по ссылке в колонке `путь` ведёт на `/p/{slug}/notes/raw?path=...`. DC отдаёт plain text (не renderуется в HTML, не парсится markdown). Браузер покажет содержимое в стандартном моноширинном виде.

Это сделано умышленно: конспекты часто содержат code-блоки, mermaid-диаграммы, frontmatter — сырой текст проще копировать в другой инструмент (Obsidian, VS Code).

Если хочется красивого markdown-render — открой файл в Obsidian (если у тебя Obsidian-vault'ом является project-папка) или в VS Code с markdown preview.

## Защита path-traversal

DC проверяет, что запрошенный `path` не выходит за пределы `learning_notes_dir`. Попытка `path=../../etc/passwd` или `path=../../config.yaml` отдаст 400.

Реализация: путь резолвится в абсолютный, и проверяется, что он начинается с `learning_notes_dir`. Поэтому в URL всегда **относительный** путь от notes_dir.

## Если конспектов нет

Если каталог пустой — увидишь текст «Конспектов нет. Они появляются после успешных self-study сессий (записываются в этот каталог).»

Когда стоит ожидать первый конспект:
- Запусти любого агента через `Start session` (см. [`rotation.md`](rotation.md)).
- Дождись завершения сессии (статус `success`).
- Обнови страницу `/notes` — увидишь.

Если сессия завершилась `success`, а конспекта нет — значит, slash-команда `/self-study` не написала файл. Проверь её реализацию в `~/.claude/commands/self-study.md`.

Если каталог `learning_notes_dir` не существует совсем — DC его не создаст автоматически. Создай вручную: `mkdir D:\Work\micode\my-app\.claude\agents\learning-notes`.
