JSON формат — flat-словарь `key: string`:

```json
{
  "common.app_name": "AI Dreaming Center",
  "navbar.all_projects": "Все проекты",
  "p.dashboard": "Панель",
  "p.metrics.success": "Успешные"
}
```

UTF-8 без BOM. **Никаких** trailing comma'ов (это JSON, не JSON5).

## Key conventions

Dot-separated namespaces:

| Префикс | Назначение |
|---|---|
| `common.*` | Общие fragments (app name, locale labels). |
| `navbar.*` | Header / nav. |
| `setup.*` | Setup wizard. |
| `projects.*` | `/projects` страница. |
| `settings.*` | Глобальные settings. |
| `p.*` | Per-project sub-страницы (`p.dashboard`, `p.live`, `p.rotation`, ...). |
| `p.<page>.*` | Sub-keys внутри страницы (например `p.metrics.success`). |
| `errors.*` | Error pages. |

Это конвенция, не enforced в коде — но check_i18n предполагает unique keys внутри обоих файлов.

## `t()` filter

Bind в `main.py:40`:

```python
def _t(key: str, locale: str | None = None, **fmt) -> str:
    return app.state.i18n.t(key, locale=locale, **fmt)

app.state.templates.env.filters["t"] = _t
```

Использование в шаблонах:

```jinja
<title>{{ "common.app_name" | t(locale=locale) }}</title>

<h1>{{ "p.rotation" | t(locale=locale) }}</h1>

<!-- с form-args: -->
<p>{{ "intro.greeting" | t(locale=locale, name=user_name) }}</p>
```

`I18n.t` ([`i18n.py:22`](../../dreaming/services/i18n.py)):

```python
def t(self, key: str, locale: str | None = None, **fmt) -> str:
    loc = locale or 'ru'
    msg = self.messages.get(loc, {}).get(key)
    if msg is None and loc != 'ru':
        msg = self.messages.get('ru', {}).get(key)
    if msg is None:
        return key
    if fmt:
        try:
            return msg.format(**fmt)
        except (KeyError, IndexError):
            return msg
    return msg
```

## Locale fallback chain

```
locale аргумент (cookie/query)
    |
    v
messages[locale][key]
    |
    v   if missing
messages['ru'][key]
    |
    v   if missing
return key as-is
```

Когда видишь raw key типа `p.dashboard` на странице — значит ключ не определён ни в RU, ни в EN. Иди добавлять.

Locale source order в роутах:

```python
locale = request.cookies.get("dc_locale", request.app.state.settings.default_locale)
```

То есть:
1. Cookie `dc_locale` (если выставлен через `POST /locale`).
2. `settings.default_locale` (default `"ru"`).

В `t()` если передан невалидный locale — fallback на 'ru'.

## CLDR Russian plurals

Russian имеет 3 plural forms (по CLDR rules):

| Категория | Условие | Примеры |
|---|---|---|
| `one` | mod10==1 && mod100!=11 | 1, 21, 31, 101, 121 |
| `few` | mod10 in {2,3,4} && mod100 not in {12,13,14} | 2, 3, 4, 22, 23 |
| `many` | (everything else) | 5..20, 25..30, 100, 111-114 |

Function ([`i18n.py:42`](../../dreaming/services/i18n.py)):

```python
def russian_plural(n: int) -> str:
    n = abs(int(n))
    mod10, mod100 = n % 10, n % 100
    if mod10 == 1 and mod100 != 11:
        return "one"
    if mod10 in (2, 3, 4) and mod100 not in (12, 13, 14):
        return "few"
    return "many"

def english_plural(n: int) -> str:
    return "one" if abs(int(n)) == 1 else "other"
```

Special cases для 11–14: «11 проектов» = many, не one. Это и ловит `mod100 != 11`.

Использование `plural()`:

```python
i18n.plural("count.runs", n, locale="ru")
# при n=1 → ищет "count.runs.one" → "1 запуск"
# при n=2 → ищет "count.runs.few" → "{n} запуска"
# при n=5 → ищет "count.runs.many" → "{n} запусков"
```

В messages JSON нужно объявить **три** ключа на русский plural и **два** на английский:

```json
{
  "count.runs.one": "{n} запуск",
  "count.runs.few": "{n} запуска",
  "count.runs.many": "{n} запусков"
}
```

В EN:

```json
{
  "count.runs.one": "{n} run",
  "count.runs.other": "{n} runs"
}
```

## Locale switching

UI: dropdown в navbar, отправляет form:

```html
<form method="post" action="/locale">
  <input type="hidden" name="next" value="{{ request.url.path }}">
  <select name="locale" onchange="this.form.submit()">
    <option value="ru">{{ "common.locale.ru" | t(locale=locale) }}</option>
    <option value="en">{{ "common.locale.en" | t(locale=locale) }}</option>
  </select>
</form>
```

Handler `POST /locale` ([`root.py:126`](../../dreaming/routes/root.py)):

```python
@router.post("/locale")
async def set_locale(request, locale: str = Form(...), next: str = Form("/")):
    if locale not in ("ru", "en"):
        locale = "ru"
    resp = RedirectResponse(url=next or "/", status_code=303)
    resp.set_cookie("dc_locale", locale, max_age=60*60*24*365,
                    httponly=False, samesite="lax")
    return resp
```

`max_age=1 год`. `httponly=False` потому что мы возможно хотим читать из JS на клиенте (на текущий момент — нет, но запас).

## check_i18n.py + smoke_i18n.py

[`scripts/check_i18n.py`](../../scripts/check_i18n.py) — verifier RU/EN parity.

Запуск:

```bash
python scripts/check_i18n.py
```

Что делает:
1. Читает оба JSON.
2. Считает ключи которые в RU но не в EN, и наоборот.
3. Если есть divergence — печатает diff и выходит с code 1.
4. Иначе exit 0.

Используй в pre-commit hook:

```bash
#!/bin/bash
python scripts/check_i18n.py || exit 1
```

[`scripts/smoke_i18n.py`](../../scripts/smoke_i18n.py) — sanity check на загрузку и plural rules:

```bash
python scripts/smoke_i18n.py
# Проверяет:
# - I18n() конструктор не падает
# - russian_plural(1) == "one"
# - russian_plural(2) == "few"
# - russian_plural(11) == "many"
# - russian_plural(21) == "one"
```

## Adding a new key

Чек-лист:

1. Открой `dreaming/i18n/messages_ru.json`. Добавь:
   ```json
   "p.heartbeat": "Heartbeat"
   ```
2. Открой `dreaming/i18n/messages_en.json`. Добавь:
   ```json
   "p.heartbeat": "Heartbeat"
   ```
3. Используй в шаблоне:
   ```jinja
   <a href="/p/{{ project.slug }}/heartbeat">{{ "p.heartbeat" | t(locale=locale) }}</a>
   ```
4. Проверь:
   ```bash
   python scripts/check_i18n.py
   ```
   Должен вернуть exit 0.

Если используешь plurals (3 RU, 2 EN форм) — добавляй сразу все, иначе fallback покажет ключ-as-is для пропущенного варианта.

### Encoding warnings

При создании / редактировании JSON файлов на Windows из PowerShell **НЕ используй** `Set-Content` — он по умолчанию пишет UTF-16. Используй:
- Edit / Write tools (UTF-8 без BOM).
- `[IO.File]::WriteAllText($path, $content, [System.Text.UTF8Encoding]::new($false))` — explicit UTF-8 без BOM.

См. [`troubleshooting.md`](../troubleshooting.md#mojibake-на-cyrillic).
