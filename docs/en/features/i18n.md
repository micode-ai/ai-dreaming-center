JSON format — flat dictionary `key: string`:

```json
{
  "common.app_name": "AI Dreaming Center",
  "navbar.all_projects": "Все проекты",
  "p.dashboard": "Панель",
  "p.metrics.success": "Успешные"

```

UTF-8 without BOM. **No** trailing commas (this is JSON, not JSON5).

## Key conventions

Dot-separated namespaces:

| Prefix | Purpose |
|---|---|
| `common.*` | Shared fragments (app name, locale labels). |
| `navbar.*` | Header / nav. |
| `setup.*` | Setup wizard. |
| `projects.*` | `/projects` page. |
| `settings.*` | Global settings. |
| `p.*` | Per-project sub-pages (`p.dashboard`, `p.live`, `p.rotation`, ...). |
| `p.<page>.*` | Sub-keys inside a page (e.g. `p.metrics.success`). |
| `errors.*` | Error pages. |

A convention, not enforced in code — but check_i18n assumes unique keys within both files.

Use in templates:

```jinja
<title>{{ "common.app_name" | t(locale=locale) }}</title>

<h1>{{ "p.rotation" | t(locale=locale) }}</h1>

<!-- with format args: -->
<p>{{ "intro.greeting" | t(locale=locale, name=user_name) }}</p>
```

`I18n.t` ([`i18n.py:22`](../../../dreaming/services/i18n.py)):

```python
def t(self, key: str, locale: str | None = None, **fmt) -> str:
    loc = locale or 'ru'
    msg = self.messages.get(loc, {}).get(key)
    if msg is None and loc != 'ru':
        msg = self.messages.get('ru', {}).get(key)
    if msg is None:
        return key
    if fmt:
        try:
            return msg.format(**fmt)
        except (KeyError, IndexError):
            return msg
    return msg
```

## Locale fallback chain

```
locale arg (cookie/query)
    |
    v
messages[locale][key]
    |
    v   if missing
messages['ru'][key]
    |
    v   if missing
return key as-is
```

When you see a raw key like `p.dashboard` on a page — the key is missing in both RU and EN. Add it.

Locale source order in routes:

```python
locale = request.cookies.get("dc_locale", request.app.state.settings.default_locale)
```

That is:
1. Cookie `dc_locale` (if set via `POST /locale`).
2. `settings.default_locale` (default `"ru"`).

In `t()`, if an invalid locale is passed — fallback to 'ru'.

## CLDR Russian plurals

Russian has 3 plural forms (per CLDR rules):

| Category | Condition | Examples |
|---|---|---|
| `one` | mod10==1 && mod100!=11 | 1, 21, 31, 101, 121 |
| `few` | mod10 in {2,3,4} && mod100 not in {12,13,14} | 2, 3, 4, 22, 23 |
| `many` | (everything else) | 5..20, 25..30, 100, 111-114 |

Function ([`i18n.py:42`](../../../dreaming/services/i18n.py)):

```python
def russian_plural(n: int) -> str:
    n = abs(int(n))
    mod10, mod100 = n % 10, n % 100
    if mod10 == 1 and mod100 != 11:
        return "one"
    if mod10 in (2, 3, 4) and mod100 not in (12, 13, 14):
        return "few"
    return "many"

def english_plural(n: int) -> str:
    return "one" if abs(int(n)) == 1 else "other"
```

Special cases for 11–14: "11 проектов" = many, not one. The `mod100 != 11` clause catches that.

Use of `plural()`:

```python
i18n.plural("count.runs", n, locale="ru")
# n=1 → looks up "count.runs.one" → "1 запуск"
# n=2 → looks up "count.runs.few" → "{n} запуска"
# n=5 → looks up "count.runs.many" → "{n} запусков"
```

In the messages JSON you must declare **three** keys for the Russian plural and **two** for English:

```json
{
  "count.runs.one": "{n} запуск",
  "count.runs.few": "{n} запуска",
  "count.runs.many": "{n} запусков"
}
```

In EN:

```json
{
  "count.runs.one": "{n} run",
  "count.runs.other": "{n} runs"
}
```

## Locale switching

UI: dropdown in the navbar, sends a form:

```html
<form method="post" action="/locale">
  <input type="hidden" name="next" value="{{ request.url.path }}">
  <select name="locale" onchange="this.form.submit()">
    <option value="ru">{{ "common.locale.ru" | t(locale=locale) }}</option>
    <option value="en">{{ "common.locale.en" | t(locale=locale) }}</option>
  </select>
</form>
```

Handler `POST /locale` ([`root.py:126`](../../../dreaming/routes/root.py)):

```python
@router.post("/locale")
async def set_locale(request, locale: str = Form(...), next: str = Form("/")):
    if locale not in ("ru", "en"):
        locale = "ru"
    resp = RedirectResponse(url=next or "/", status_code=303)
    resp.set_cookie("dc_locale", locale, max_age=60*60*24*365,
                    httponly=False, samesite="lax")
    return resp
```

`max_age=1 year`. `httponly=False` because we may want to read from client-side JS (currently — no, but as a safety net).

## check_i18n.py + smoke_i18n.py

[`scripts/check_i18n.py`](../../../scripts/check_i18n.py) — RU/EN parity verifier.

Run:

```bash
python scripts/check_i18n.py
```

What it does:
1. Reads both JSONs.
2. Counts keys that exist in RU but not in EN, and vice versa.
3. If there's divergence — prints the diff and exits with code 1.
4. Otherwise exit 0.

Use in a pre-commit hook:

```bash
#!/bin/bash
python scripts/check_i18n.py || exit 1
```

[`scripts/smoke_i18n.py`](../../../scripts/smoke_i18n.py) — sanity check for load + plural rules:

```bash
python scripts/smoke_i18n.py
# Verifies:
# - I18n() constructor doesn't crash
# - russian_plural(1) == "one"
# - russian_plural(2) == "few"
# - russian_plural(11) == "many"
# - russian_plural(21) == "one"
```

## Adding a new key

Checklist:

1. Open `dreaming/i18n/messages_ru.json`. Add:
   ```json
   "p.heartbeat": "Heartbeat"
   ```
2. Open `dreaming/i18n/messages_en.json`. Add:
   ```json
   "p.heartbeat": "Heartbeat"
   ```
3. Use it in the template:
   ```jinja
   <a href="/p/{{ project.slug }}/heartbeat">{{ "p.heartbeat" | t(locale=locale) }}</a>
   ```
4. Verify:
   ```bash
   python scripts/check_i18n.py
   ```
   Must return exit 0.

If you use plurals (3 RU, 2 EN forms) — add them all, otherwise the fallback returns the key-as-is for the missing variant.

### Encoding warnings

When creating / editing JSON files on Windows from PowerShell **do NOT use** `Set-Content` — its default writes UTF-16. Use:
- Edit / Write tools (UTF-8 without BOM).
- `[IO.File]::WriteAllText($path, $content, [System.Text.UTF8Encoding]::new($false))` — explicit UTF-8 without BOM.

See [`troubleshooting.md`](../troubleshooting.md#mojibake-on-cyrillic).
